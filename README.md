# Hackathon Azzas

Documentação geral do **Hackathon Azzas** — onboarding e guia dos pilares. Este repositório
também já vem com um **set de skills pré-configurado** para o Claude: ao clonar e abrir o Claude
Code aqui, o time começa com tudo no lugar (regras, guias de pilar, acesso a dados).

## Comece por aqui

➡️ **[Onboarding](docs/onboarding/)** — clone, setup, conexão ao MCP e regras do jogo (5 min).

## O que tem no repo

```
hackathon-azzas/
├── CLAUDE.md              # instruções carregadas pelo Claude ao abrir o repo
├── .claude/skills/        # set de skills pré-configurado (ver abaixo)
└── docs/
    ├── onboarding/        # guia de entrada
    ├── pilares/           # os 4 pilares e seus cases
    └── recursos/          # referência do MCP de dados, materiais de apoio
```

## Os 4 pilares

| Pilar | Case |
|---|---|
| **Criativo** | Coleção cápsula de até 30 produtos (briefing, produtos, ficha de repasse, showroom). |
| **Compras** | Distribuição de peças da Fábula para a coleção INV26. |
| **Eficiência Operacional** | Automatizar um fluxo operacional do dia a dia. |
| **Receita & Clientes** | Nova forma de segmentar a base de clientes. |

Detalhes: [`docs/pilares/`](docs/pilares/).

## Skills incluídas

| Skill | Para quê |
|---|---|
| `azzas-dados` | Como usar o MCP de dados (escopo, roteamento, PII, tabelas). |
| `azzas-contexto-publico` | Contexto institucional público do grupo (marcas, estrutura, história). |
| `consulting-storytelling` | Estruturar a apresentação final no padrão de consultoria. |
| `azzas-identidade-visual` | Design system da Azzas (paleta, tipografia, componentes) para padronizar entregas visuais. |
| `pilar-criativo` · `pilar-compras` · `pilar-eficiencia-operacional` · `pilar-receita-clientes` | Guia por pilar. |
